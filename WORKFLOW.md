# 微信公众号 AI 写作系统 — 核心工作流与脉络图

## 一、系统定位

这是一个**有灵魂、会进化的微信公众号内容创作系统**，不是简单的模板填充工具。支持五类文章写作、范文学习、素材沉淀、质量闸门、合规检查、数据复盘、多平台改写和自我进化。

## 四、核心写作工作流（第零步~第十步，共十一步）

```
第零步：唤醒人格
  │  读取 soul.md → personality.md → feedback.md → self-evolution.md
  │  加载/推荐风格（--style 或自动推荐 Top3）
  ▼
第一步：需求确认
  │  解析参数（类型+主题），智能推断缺失参数
  │  三级引导策略：零追问直接干 / 轻确认 / 简单追问
  │  绝不追问：赛道、读者、字数、风格、发布时间、情绪基调
  ▼
第二步：加载写作规范
  │  style-guide + formatting + 对应模板
  │  + 记忆库（金句/标题/选题/素材/对标/排期）
  │  + 素材自学习（从 materials 提炼金句和标题候选）
  ▼
第三步：内心独白（仅写文章时）
  │  以"这个人"的身份思考选题，100-200字，输出给用户看
  │  体现当前性格维度，影响后续文章角度和风格
  ▼
第四步：构建文章大纲
  │  标题(≥5备选) + 开头Hook + 正文结构 + 结尾策略 + 预估字数
  │  ──→ 等待用户确认 ←──
  ▼
第五步：撰写全文
  │  标题规则（≤20字，数字/疑问/对比技巧）
  │  开头规则（前3行定生死，禁止自我介绍和长铺垫）
  │  正文规则（段落≤4行，每300-500字小标题，金句10%）
  │  结尾规则（总结+互动引导+自然引导关注）
  │  核心：用户输入是原料不是初稿，文字重合度须≤30%
  ▼
第五步半：配图规划与生成
  │  确定位置（按文章类型策略）
  │  → 选择风格（16种，按类型自动匹配）
  │  → 生成提示词（内容描述+构图指令+风格锚词）
  │  → 输出方案供用户确认
  │  → 调用 scripts/generate_images.py（云雾API并发生成）
  ▼
第六步：风险检测（基于微信官方规范）
  │  10项必检：安全红线/不实信息/标题风险/诱导行为/版权原创
  │           /广告营销/特殊领域/推荐规范/封面图/敏感词
  │  风险评级：🟢安全 🟡注意 🟠警告 🔴危险
  ▼
第七步：自检优化
  │  20+项检查清单
  │  + 原创度/AI味/人味 量化闸门（scripts/originality_quality_gate.py）
  │  不达标 → 定向重写（最多3轮，只改形式不改内容）
  │  去AI味优先级：删套话 → 拉句长差异 → 替换空词 → 增立场句 → 清来源痕迹
  ▼
第八步：输出最终稿
  │  标题(≥5) + 正文(Markdown+配图) + 摘要(≥5) + 标签(3-5)
  │  + 配图信息 + 风险评估 + 交付回执
  │  输出模式：md(默认) / json / both / wechat
  │  落地路径：outputs/{风格名}/{日期}-{类型}-{主题}/
  ▼
第九步：记忆存档与人格成长（自动执行）
   ├─ 金句 → golden-sentences.md
   ├─ 选题 → topics.md（+衍生3-5个新选题）
   ├─ 素材 → materials.md
   ├─ 标题偏好 → titles.md
   ├─ 人格微调 → personality.md（±0.5/次）
   ├─ 心境日记 → personality.md
   └─ 用户反馈 → feedback.md + self-evolution.md
  ▼
第十步：上传公众号草稿箱（用户触发，可选）
   ├─ 配置检查（config/wechat.local.yaml，兼容 config/wechat.yaml）
   ├─ 确认上传内容（标题/摘要/封面图/作者）
   ├─ MD→HTML + 图片上传 + 推送草稿箱
   └─ 反馈结果（成功：草稿ID / 失败：错误排查）
```

## 五、辅助工作流

### 5.1 范文学习流程

```
用户提供范文 + 作者名（必填）
  │
  ├─ 保存原文 → learning/samples/[作者名]/
  │
  ├─ 类型自动识别（评分矩阵：主类型+副类型+混合比例+置信度）
  │
  ├─ 8维度深度拆解
  │   结构 / 排版 / 语气 / 文风 / 金句 / 论证 / 互动 / 标点怪癖
  │
  ├─ 素材提取（自动分类存入 memory/materials.md）
  │   案例故事 / 数据论据 / 金句观点 / 结构模板 / 标题技巧 / 类比比喻
  │
  ├─ 风格冲突检测（对比 personality.md）
  │   ├─ 无冲突 → 正常人格消化（±0.5）
  │   ├─ 有冲突 → 风格只存 styles/，人格不变
  │   └─ 已锁定 → 跳过人格更新
  │
  └─ 生成/更新风格档案 → styles/[作者名].md
```

### 5.2 数据复盘闭环

```
录入效果数据（标题+阅读量+互动指标）
  → 存入 memory/performance.md
  → 对比基准线，自动判断好/一般/差
  → 诊断问题（按优先级列3条）
  → 输出可执行动作 + 回测时间点
  → 积累10+篇 → 自动更新高效模式 + 基准线
  → 连续3篇差 → 触发策略调整预警
```

### 5.3 选题库管理

```
输入来源                       处理                        输出
─────────                    ────                       ────
对标文章/素材  ──→  批量提取选题  ──→  memory/topics.md (CSV)
已完成文章    ──→  提取+衍生3-5个  ──→  按赛道分类，含爆款潜力评分
选题库整理    ──→  去重/评分/排序  ──→  待写选题按潜力排序
```

### 5.4 对标账号分析

```
用户提供账号名 + 20篇文章样本
  → 分组验证（实验组10篇 + 对照组10篇）
  → 提炼策略 → 对照验证 → 保留稳定策略
  → 存入 memory/benchmarks.md
```

### 5.5 多平台改写

```
公众号原文 ──→ 小红书笔记（图文种草风格）
           ──→ 知乎回答（深度分析风格）
           ──→ 短视频口播文案（口语化节奏）
```

### 5.6 内容日历

```
历史发布节奏 / 用户设定
  → 生成周排期 / 月规划
  → 节假日热点规划 + 内容类型搭配
  → 存入 memory/calendar.md
```

## 七、质量闸门（输出前必须全部通过）

| 指标 | 阈值 | 检测工具 |
|------|------|---------|
| 原创度 `originality_score` | ≥ 70 | `scripts/originality_quality_gate.py` |
| AI味 `ai_tone_score` | ≤ 30 | 同上 |
| 人味 `humanity_score` | ≥ 60 | 同上 |
| 来源痕迹 `source_trace_hits` | = 0 | 同上 |
| 模板句比例 `template_sentence_ratio` | ≤ 12% | 同上 |
| 句长变异系数 `sentence_cv` | ≥ 0.35 | 同上 |
| 段长变异系数 `paragraph_cv` | ≥ 0.45 | 同上 |
| 素材重合度 | ≤ 30% | 正文 vs 用户原始输入 |

不达标处理：定向重写（最多3轮） → 3轮后仍不达标 → 输出失败回退模板

## 八、规则优先级（从高到低）

```
1. core/soul.md 红线        → 绝对优先，任何风格都不能突破底线
2. styles/[name].md         → --style 时覆盖表达层（语气/结构/文风/金句）
3. core/personality.md      → 风格未覆盖的维度，用当前人格
4. rules/ + templates/      → 通用规范兜底
```

**关键原则**：
- `--style` 改变"怎么写"，不改变"我是谁"
- 互动方式始终由 personality.md 决定，风格文件中的互动字段一律忽略
- 全局偏好（可读性、短句密度、合规）优先于风格偏好

## 九、文件职责边界（防止修改漂移）

| 内容 | 归属文件 | 不可重复出现在 |
|------|---------|--------------|
| 总控流程（第零步~第十步） | SKILL.md | prompts/prompt.md |
| 参数推断规则、引导策略 | SKILL.md | prompts/prompt.md |
| 五类文章生文提示词 | prompts/prompt.md | SKILL.md |
| 写作风格规则 | rules/style-guide.md | SKILL.md / prompts/prompt.md |
| 风险检测详细规则 | rules/risk-check.md | SKILL.md（只放入口） |
| 素材提取分类规范 | materials.md（格式）+ SKILL.md（步骤）+ prompt.md（提示词） | 三者分工不重复 |
| 自创金句 | golden-sentences.md | 不存 materials.md |
| 外部素材 | materials.md | 不存 golden-sentences.md |

**修改原则**：改规则时只改归属文件，另一个文件通过引用指向，不复制内容。
