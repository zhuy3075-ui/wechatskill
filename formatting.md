# 微信公众号排版规范

## 一、基础排版原则

微信公众号文章在手机上阅读，排版的核心目标是：**降低阅读压力，提升阅读体验**。

### 关键数据
- 手机屏幕宽度有限，每行约显示 16-18 个汉字
- 读者平均阅读时长 3-5 分钟
- 超过 60% 的读者不会滑到文章底部

---

## 二、段落规范

### 段落长度
- **每段不超过 4 行**（手机显示行数）
- 约 60-80 字为一段
- 重要观点独立成段，哪怕只有一句话

### 段间距
- 段落之间空一行（即两个换行符）
- 小标题前空一行，后不空行
- 图片前后各空一行

### 缩进
- **不使用首行缩进**（微信排版传统）
- 所有段落顶格写

---

## 三、标题层级

微信文章建议最多使用 3 级标题：

```markdown
# 文章大标题（仅用一次，作为文章标题）

## 一级小标题（主要段落分隔）

### 二级小标题（段落内细分，较少使用）
```

### 小标题规范
- 小标题要有信息量，不要用「第一点」「其次」这种
- 好的小标题：「为什么 90% 的人坚持不下去」
- 差的小标题：「关于坚持的问题」
- 每 300-500 字设置一个小标题
- 小标题可以用 **加粗** 替代 Markdown 标题语法

---

## 四、强调与高亮

### 加粗（Bold）
- 用于**核心观点**和**关键信息**
- 每段最多加粗 1-2 处，不要滥用
- 加粗内容控制在 10 字以内

### 引用块（Blockquote）
用于以下场景：
- 金句/名言
- 重要总结
- 他人观点

```markdown
> 真正的自由，不是想做什么就做什么，而是不想做什么就可以不做什么。
```

### 分隔线
- 用于主要章节之间的分隔
- 全文使用不超过 3 条
- 使用 `---` 表示

---

## 五、列表使用

### 无序列表
适用于并列信息、注意事项：
```markdown
- 第一点内容
- 第二点内容
- 第三点内容
```

### 有序列表
适用于步骤、排名：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 列表规范
- 每个列表项不超过 2 行
- 列表项数控制在 3-7 个
- 超过 7 个考虑分组

---

## 六、图片与视觉元素

### 图片建议
- 每 500-800 字插入一张图
- 图片宽度建议 1080px
- 使用 `[图片描述]` 标记图片位置

### Emoji 使用
- 适量使用增加亲和力
- 全文不超过 5 个 emoji
- 推荐用在小标题前：📌 重点提醒
- 不要用 emoji 替代文字表达

### 常用装饰符号
```
📌 重点/提醒
💡 技巧/灵感
⚠️ 注意/警告
✅ 正确做法
❌ 错误做法
👉 引导/指向
```

## 八、Markdown 输出注意事项

由于公众号编辑器需要从 Markdown 转换，注意以下事项：

### 兼容性
- 使用标准 Markdown 语法
- 避免使用 HTML 标签
- 表格在手机上显示效果差，尽量避免使用
- 代码块在非技术类文章中避免使用

### 复制粘贴友好
- 输出的文章应该可以直接复制到 Markdown 编辑器
- 再通过 Markdown 编辑器（如墨滴、微信 Markdown 编辑器）转换为公众号格式
- 确保所有排版在转换后保持正确

## 九、字数参考

| 文章类型 | 建议字数 | 阅读时间 |
|---------|---------|---------|
| 干货型 | 2000-3000 字 | 5-8 分钟 |
| 观点型 | 1500-2500 字 | 4-6 分钟 |
| 故事型 | 2000-3500 字 | 5-9 分钟 |
| 清单型 | 1500-2500 字 | 4-6 分钟 |
| 热点型 | 1000-2000 字 | 3-5 分钟 |

---

## 十、最终检查清单

发布前逐项检查：

- [ ] 段落长度是否合规（≤4 行）
- [ ] 是否有足够的小标题分段
- [ ] 加粗是否适度（没有大段加粗）
- [ ] 引用块使用是否恰当
- [ ] 图片位置标注是否清晰
- [ ] 没有使用首行缩进
- [ ] 段间距是否统一
- [ ] Emoji 数量是否合理（≤5 个）
- [ ] 整体视觉是否清爽、不压抑
