# 观点与立场库

> 记录从用户提供的文章和对话中提取的核心观点、用户立场、价值判断。
> 跨会话保留，用于理解用户的思想倾向，让后续创作更贴合用户的真实态度。
>
> **与其他文件的分工**：
> - `viewpoints.md`（本文件）：用户的核心观点、立场、价值判断
> - `feedback.md`：用户对文章的修改偏好（写作层面）
> - `materials.md`：外部素材（案例、数据、他人观点）

## 使用说明

### 写入时机

1. **用户提供文章时**：提取文章中的核心观点，标注是否代表用户立场
2. **对话中用户表达观点时**：捕捉用户对某个话题的态度和判断
3. **用户修改文章观点时**：记录用户删除/修改了哪些观点，推断立场

### 观点格式

```
- [V-001] [观点内容（1-2句话）]
  来源：用户提供文章《标题》/ 对话 / 用户修改反馈
  话题：[所属话题领域]
  立场强度：强认同 / 认同 / 中立引用 / 不确定
  标签：#关键词1 #关键词2
  添加日期：YYYY-MM-DD
  关联金句：G-XXX（如有对应金句）
```

**观点ID 规则**：`V-XXX`，三位递增编号，删除后不回收。

### 立场强度判定

| 强度 | 判定依据 |
|------|---------|
| 强认同 | 用户明确说"我认为""我一直觉得"，或多次在不同场景重复该观点 |
| 认同 | 用户在文章中作为主论点使用，或对话中表示赞同 |
| 中立引用 | 用户提供的文章中包含，但未明确表态是否认同 |
| 不确定 | 仅出现一次，无法判断用户态度，待后续对话确认 |

### 立场升级规则

- 同一观点在 3 次以上对话/文章中出现 → 自动升级为「强认同」
- 用户明确否定某观点 → 标记为「反对」并记录原因
- 两个观点互相矛盾 → 标记冲突，下次创作时主动询问用户

### 容量与维护

- 上限 150 条
- 「不确定」立场超过 3 个月未确认 → 移入底部「待确认区」
- 每积累 30 条 → 自动归纳用户在各话题领域的立场倾向总结
