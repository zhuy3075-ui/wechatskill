# 配图风格库

> 按文章类型定义默认图片风格和可选风格。
> 每个风格包含「风格锚词」，作为所有提示词的统一后缀，确保同一篇文章内视觉一致。

---

## 一、风格定义

### 扁平插画（信息图风）

```yaml
id: flat-illustration
名称: 扁平插画
适用类型: [干货]
描述: 简洁清晰的扁平设计插画，适合知识类、教程类文章，视觉信息量大但不喧宾夺主
风格锚词: 扁平插画风格，简洁干净，柔和渐变配色，无描边，白色背景，现代设计感，矢量图质感，信息图风格
构图偏好: 居中构图，留白充足，元素清晰
色调: 明亮柔和，低饱和渐变色
```

## 三、配图位置策略（B+C 混合，按文章类型）

| 文章类型 | 策略 | 配图数量 | 位置逻辑 |
|---------|------|---------|---------|
| 干货型 | B 为主 + C 辅助 | 3-5 张 | 每个方法论/工具小节之间 + 关键步骤处 |
| 观点型 | C 为主 | 2-3 张 | 核心论点转折处 + 情绪高点 + 结论前 |
| 故事型 | B+C 混合 | 3-4 张 | 场景转换处 + 情感高潮 + 反转点 |
| 清单型 | B 为主 | 4-6 张 | 每 2-3 个清单项之间 + 总览图 |
| 热点型 | C 为主 | 2-3 张 | 事件关键节点 + 观点转折处 |

### 通用规则

- 封面图：每篇必生成 1 张
- 相邻两张图间距不少于 300 字
- 图片不放在小标题正下方（先有一段文字再插图）
- 图片放在段落之间，不打断段落
- 文章结尾互动引导部分不插图

## 四、提示词生成规则

每张配图的提示词由三部分拼接：

```
[内容描述] + [构图指令] + [风格锚词]
```

### 内容描述生成规则
1. 分析配图位置前后各 200 字上下文
2. 提取该段落的核心意象（人物/场景/物品/概念）
3. 转化为可视化的具体画面描述
4. 避免抽象概念，转为具体场景（如 "时间管理" → "一个人在整洁书桌前写计划表"）

### 构图指令对照表
| 图片位置 | 构图建议 |
|---------|---------|
| 封面图 | 横构图，主体突出，有视觉焦点，适合裁切 |
| 头图/氛围图 | 横构图，全景或远景，氛围优先 |
| 段间配图 | 横构图，中景，均衡构图 |
| 情绪高点 | 横构图，特写或中近景，聚焦细节 |
| 关键转折 | 横构图，对比构图或引导线构图 |

### 提示词示例

```
内容描述：一个年轻人在整洁的书桌前认真写计划表，桌上放着一杯咖啡和几本书，窗外阳光照进来
构图指令：横构图，中景，自然光线，浅景深
风格锚词：暖调生活摄影，柔和自然光，温暖色调，轻微胶片质感，真实感，有生活气息

完整提示词：
一个年轻人在整洁的书桌前认真写计划表，桌上放着一杯咖啡和几本书，窗外阳光照进来，横构图，中景，自然光线，浅景深，暖调生活摄影，柔和自然光，温暖色调，轻微胶片质感，真实感，有生活气息
```

## 五、风格扩展

> 用户可以通过范文学习提取对标账号的配图风格，新增自定义风格。
> 自定义风格按相同格式添加到本文件的「一、风格定义」中。

### 自定义风格模板

```yaml
id: custom-style-name
名称: 你的风格名
适用类型: [干货, 观点]
描述: 一句话描述这个风格的视觉特点
风格锚词: 用逗号分隔的关键词，作为所有提示词的统一后缀
构图偏好: 这个风格偏好的构图方式
色调: 这个风格的主色调方向
```
