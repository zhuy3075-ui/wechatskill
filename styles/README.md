# 风格文件说明

`styles/` 目录中的每个 `*.md` 都可被 `scripts/style_recommender.py` 自动识别并参与推荐。

建议每个风格文件包含以下字段（可选，但越完整推荐越准）：

```markdown
# 犀利派

语气：直接、锋利、少铺垫
结构：结论前置 -> 反证 -> 行动建议
适用：观点、热点
关键词：反常识、冲突、判断、立场
全局偏好：遵守可读性与密度上限（短句/标点/破折号）
风格偏好：口头禅、怪癖标点、俚语方言策略（仅本风格生效）
短句策略：关键句可独立成段，避免连续碎句
标点策略：破折号适中，解释优先冒号/逗号
俚语方言：按场景启用（非必需不加）
```

说明：
- `适用` 支持多个类型：`干货/观点/故事/清单/热点`
- `关键词` 支持空格、逗号、`#` 混写
- `全局偏好` 是所有风格都要遵守的硬规则（可读性优先）
- `风格偏好` 是当前风格独有偏好，风格间互不继承
- `短句策略/标点策略/俚语方言` 建议显式填写，便于稳定执行
- 未写字段时，脚本会自动从正文提取关键词做降级推荐
